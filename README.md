# CryptoAnalysis - Hyperliquid 交易策略分析系统

基于 Hyperliquid 链上透明数据的量化交易策略研究项目。

## 🎯 核心策略：反脆弱策略 (Anti-Fragile Strategy)

> **最新文档：** [docs/ANTI_FRAGILE_STRATEGY.md](docs/ANTI_FRAGILE_STRATEGY.md)

### 核心理念

```
Alpha = 聪明钱正向跟单 (30-40%) + 脆弱地址反向操作 (50-60%)
```

| 策略 | 占比 | 原理 |
|------|------|------|
| **聪明钱正向跟单** | 30-40% | 跟随正确的人 |
| **脆弱地址反向** | 50-60% | 利用散户亏损的统计优势 |
| **现金/观望** | 10-20% | 信号不明确时 |

### 关键发现

1. **HyperBot 的"聪明钱"只是余额 >$10k 的地址**，真正的分类必须自建
2. **脆弱地址反向占比更高**，因为散户亏损率 70-90%，样本量大、信号稳定
3. **数据架构**：HyperBot 提供候选池，Hyperliquid 提供核心数据，自建评分模型

## 📊 数据架构

### API 数据来源

| 数据源 | 角色 | 提供数据 |
|--------|------|---------|
| **HyperBot API** | 辅助 | 候选地址池（余额>$10k）、爆仓热力图、鲸鱼多空比 |
| **Hyperliquid API** | 核心 | 持仓详情、交易历史、PnL曲线、市场数据 |
| **自建计算** | 核心 | 胜率、杠杆、清算次数、评分模型、分类逻辑 |

### 数据流程

```
HyperBot /smart/find
        │
        ▼ 候选地址列表
┌───────────────────────────────────┐
│     Hyperliquid API 逐个查询      │
│  ├── clearinghouseState (持仓)    │
│  ├── userFills (交易历史)         │
│  └── userPortfolio (PnL曲线)      │
└───────────────────────────────────┘
        │
        ▼ 特征计算
┌───────────────────────────────────┐
│         自建评分模型              │
│  ├── 聪明钱评分 (盈利+风控+稳定)  │
│  └── 脆弱评分 (风险+亏损+心态)    │
└───────────────────────────────────┘
        │
        ▼
  聪明钱池 / 脆弱地址池 → 信号生成
```

## 🏗️ 地址分类模型

### 聪明钱评分 (≥65分入池)

```
总分 = 盈利能力(40) + 风控能力(30) + 稳定性(30)

盈利能力: 胜率 + 盈亏比 + 总PnL%
风控能力: 最大回撤 + 实际杠杆 + 止损使用率
稳定性:   无单笔暴利 + 交易频率稳定 + 存活时长
```

### 脆弱地址评分 (≥55分入池)

```
总分 = 风险行为(40) + 亏损特征(35) + 心态特征(25)

风险行为: 实际杠杆 + 仓位利用率 + 单币种集中度
亏损特征: 胜率低 + 清算次数 + 总PnL%亏损
心态特征: 浮亏加仓率 + 死扛指数 + 追涨杀跌
```

### 分类矩阵

| | 脆弱评分 <40 | 脆弱评分 40-55 | 脆弱评分 ≥55 |
|--|-------------|---------------|-------------|
| **聪明≥75** | 🟢 聪明钱 | ⚠️ 观察 | ❌ 异常 |
| **聪明65-75** | ⚠️ 观察 | ⚠️ 观察 | 🔴 脆弱 |
| **聪明<65** | 中性 | 🔴 脆弱 | 🔴 高危脆弱 |

## 🛡️ 风控体系

### 熔断机制

| 触发条件 | 动作 |
|---------|------|
| 日亏 2% | 降低新开仓权重 50% |
| 日亏 3% | 停止新开仓 |
| 日亏 4% | 减仓 30% |
| 日亏 5% | 熔断，暂停 4h |
| 周亏 10% | 暂停 24h + 人工复核 |
| BTC 24h 跌 >15% | 全策略暂停 + 减仓 50% |

### 仓位管理

| 规则 | 限制 |
|------|------|
| 单信号最大仓位 | 5% |
| 多信号共振最大 | 15% |
| 总敞口上限 | 50% |
| 单地址权重 | ≤2% |
| 单币种敞口 | ≤15% |
