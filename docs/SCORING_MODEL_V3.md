## 因子一：补仓模式（15 分）

### 计算逻辑

**订单周期识别：**
- 按 coin + 方向，时间排序遍历 hl_fills
- 找到 start_position = 0 的 Open → 订单周期起点
- start_position > 0 的 Open 跳过，继续找下一个起点

**补仓计数规则：**
- Open 之后无 Close → 补仓计数 +1
- 遇到 Close → 重置补仓计数为 0
- 补仓次数 = 同一持仓周期内 Open 次数 - 1

**做T识别：**
- 同 coin + 同方向，订单周期内 Open → Close → Open = 做T 1次
- 做T累计总次数 > 15 → 0分，剔除储备池

**平均补仓次数：**
- 统计所有订单周期的补仓次数
- 取平均值，四舍五入为整数

**规律性补仓识别（等差）：**
- 连续 3 次以上加仓，价格间距相等 → 判定为规律性补仓
- 规律性补仓不计入补仓次数统计

### 评分档位

| 平均补仓次数 | 得分 |
|------------|------|
| 0（无补仓） | 15分 |
| 1 ~ 4 | 11分 |
| 5 ~ 9 | 7分 |
| 10 ~ 15 | 4分 |
| > 15（非规律性） | 2分 |
| > 15（等差规律性） | 0分，剔除 |
| 做T > 15次 | 0分，剔除 |

## 因子二：综合盈亏特征（20 分）

### 计算逻辑

**胜率：**
```
win_rate = COUNT(closed_pnl > 0) / COUNT(*) × 100
数据来源：hl_fills 中所有 Close 记录（做T仓位剔除）
```

**盈亏比：**
```
profit_loss_ratio = AVG(盈利记录 closed_pnl) / ABS(AVG(亏损记录 closed_pnl))
盈亏比 > 1.0 → 亏得少（平均盈利 > 平均亏损）
盈亏比 ≤ 1.0 → 亏得多（平均盈利 ≤ 平均亏损）
```

### 评分档位（联合查表）

| 胜率区间 | 盈亏比 > 1.0（亏更少） | 盈亏比 ≤ 1.0（亏更多） |
|---------|-------------------|-------------------|
| < 35% | 15分 | 18分 |
| 35% ~ 40% | 10分 | 14分 |
| 40% ~ 45% | 6分 | 10分 |
| 45% ~ 50% | 2分 | 6分 |
| ≥ 50% | 0分 | 2分 |

> 注：示例文档中"胜率38%，盈亏比1.8→14分"为笔误，正确为10分。

## 因子三：清算得分（25 分）

### 计算逻辑

**清算识别（两种）：**
1. 官方清算：`dir LIKE 'Liquidat%'`
2. 近似清算：
   - `sz ≈ |start_position|`（差异 < 2%）
   - `closed_pnl < 0`
   - `|closed_pnl| / (sz × px) ≥ 50%`

**统计口径：历史总清算次数**（非每月）

### 评分档位

| 历史总清算次数 | 得分 |
|-------------|------|
| ≥ 3 | 25分 |
| = 2 | 18分 |
| = 1 | 12分 |
| 0 | 0分 |

## 因子四：追涨杀跌率 + 亏损集中度（20 分）

### 4.1 追涨杀跌率（15 分）

**计算逻辑（无需K线）：**

按订单周期统计，一个周期 = 从建仓到完全平仓：

- **做多加仓**：加仓价 > 当前持仓均价 → 追涨
- **做空加仓**：加仓价 < 当前持仓均价 → 杀跌
- 一个订单周期内有追涨或杀跌行为 → 该周期计为情绪化 1 次（不管期间追了几次）

```
追涨杀跌率 = 有情绪化加仓的订单周期数 / 总订单周期数 × 100%
```

**持仓均价计算：**
```
均价 = (已有仓位 × 原均价 + 本次 sz × px) / (已有仓位 + sz)
```

### 评分档位

| 追涨杀跌率 | 得分 |
|-----------|------|
| < 20% | 0分 |
| 20% ~ 35% | 6分 |
| 35% ~ 45% | 12分 |
| 45% ~ 55% | 16分 |
| > 55% | 20分 |

---

### 4.2 亏损集中度（5 分）

**计算逻辑：**
- 统计币种范围：BTC、ETH、SOL、DOGE、XRP、ADA、HYPE、BCH、BNB
- 按币种汇总亏损金额（closed_pnl < 0）
- 找出亏损占比最高的单一币种

```
亏损集中度 = 单一币种亏损额 / 总亏损额 × 100%
```

### 评分档位

| 单一币种亏损占比 | 得分 |
|---------------|------|
| < 30% | 0分 |
| 30% ~ 70% | 2分 |
| > 70% | 5分 |

**因子四总分 = 追涨杀跌率得分 + 亏损集中度得分（上限 20 分）**

## 因子六：平均持仓时长（5 分）

### 计算逻辑

```
对每笔已平仓交易：持仓时长 = 平仓时间 - 开仓时间
平均持仓时长 = 所有仓位持仓时长的算术平均值
数据来源：hl_fills 按订单周期计算
```

### 评分档位

| 平均持仓时长 | 得分 |
|------------|------|
| < 6小时 | 0分 |
| 6 ~ 24小时 | 1分 |
| 24 ~ 72小时 | 3分 |
| 72小时 ~ 7天 | 4分 |
| > 7天 | 5分 |
