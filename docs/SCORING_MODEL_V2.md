# 脆弱地址评分模型 v2.1

> 更新时间：2026-04-13 19:00  
> 设计理念：从"固定扣分制"转向"动态加减分制"，满分不设上限，支持负分

---

## 核心变更（v2.1 vs v2.0）

**v2.1 新增**（2026-04-13 19:00）：
- 新增**最长连续亏损笔数**指标（5分）
- 连续亏损识别改为**方案D**（时间窗口 + 方向合并），更准确
- 心态特征从 20分 → **25分**
- 理论总分从 ~120 → **~125分**

---

## 评分体系

### 总分公式

```
总分 = 风险行为(35) + 亏损特征(30) + 心态特征(25) + 加仓/做T(±15)
```

理论范围：约 -10 ~ 125 分

---

## 一、风险行为（35分）

### 1.1 账户整体杠杆（15分）

**数据来源**：`hl_position_snapshots` 历史均值
```sql
avg_leverage = AVG(total_ntl_pos / account_value)
```

| 杠杆倍数 | 得分 |
|---------|------|
| ≥10x | 15分 |
| ≥5x | 10分 |
| ≥3x | 6分 |
| ≥2x | 3分 |
| <2x | 0分 |

---

### 1.2 仓位利用率（10分，含补仓修正）

**数据来源**：`hl_position_snapshots` 历史均值
```sql
avg_margin_utilization = AVG(total_margin_used / account_value)
```

**基础得分**：

| 利用率 | 得分 |
|--------|------|
| ≥90% | 10分 |
| ≥80% | 7分 |
| ≥60% | 4分 |
| ≥40% | 2分 |
| <40% | 0分 |

**补仓修正**（has_refill_behavior = True）：

- 利用率 ≥80% 且有补仓 → 得分 -3
- 利用率 ≥60% 且有补仓 → 得分 -2

> 逻辑：有补仓空间且实际使用的人，初始高仓位风险相对可控

## 二、亏损特征（30分）

### 2.1 胜率（15分）

**数据来源**：`hl_fills` Close 记录
```sql
win_rate = COUNT(closed_pnl > 0) / COUNT(*) × 100
```

| 胜率 | 得分 |
|------|------|
| <30% | 15分 |
| <35% | 12分 |
| <40% | 8分 |
| <45% | 4分 |
| ≥45% | 0分 |

---

### 2.2 清算次数/月（10分）

**数据来源**：`hl_fills` 官方清算 + 近似清算

**清算识别规则**：

1. **官方清算**：`dir LIKE 'Liquidat%'`
2. **近似清算**：
   - `sz ≈ |start_position|`（差异 < 2%）
   - `closed_pnl < 0`
   - `|closed_pnl| / (sz × px) ≥ 50%`（亏损占名义价值50%以上）

```sql
liquidation_per_month = (官方清算 + 近似清算) / 活跃月数
```

| 清算/月 | 得分 |
|---------|------|
| ≥3 | 10分 |
| ≥2 | 7分 |
| ≥1 | 4分 |
| <1 | 2分 |
| 0 | 0分 |

|--------|------|

| ≥ 0 | 0分 |

## 三、心态特征（25分）

### 3.1 连续亏损后加仓（10分）

**数据来源**：`hl_fills` 按时间排序

**识别逻辑（方案D，v2.1改进）**：
- 同 coin + 同方向（Long/Short）+ 1小时内的连续 Close，算同一次亏损事件
- 中间有盈利 Close，归零计数
- 连续 N 次亏损事件后，下一笔是 `Open` → 计为连续亏损后加仓

| 次数 | 得分 |
|------|------|
| ≥3 | 10分 |
| ≥2 | 6分 |
| ≥1 | 3分 |
| 0 | 0分 |

> 赌徒心态最直接的体现

**改进原因**：
- 旧版：每笔 Close 都独立计数，会把"同一次做错方向的分批止损"算成多次亏损
- 新版：1小时内同方向的连续 Close 合并为一次亏损事件，更准确

---

### 3.2 追涨杀跌率（5分）

**数据来源**：币种集中度（简化版）

| 集中度 | 得分 |
|--------|------|
| ≥90% | 5分（追单一标的） |
| ≥70% | 3分 |
| <70% | 0分 |

> 完整版需要 K 线数据，当前用币种集中度替代

---

### 3.3 最长连续亏损笔数（5分）🆕

**数据来源**：`hl_fills` 按时间排序，统计最长连续亏损记录

**计算逻辑（方案D）**：
- 同 coin + 同方向（Long/Short） + 1小时内的连续 Close，算同一次亏损事件
- 中间有盈利 Close，归零计数
- 记录历史上最长的连续亏损事件数

| 最长连亏笔数 | 得分 |
|--------------|------|
| ≥10 | 5分 |
| ≥7 | 3分 |
| ≥5 | 1分 |
| <5 | 0分 |

> 反映心态崩溃程度，连续亏 10 笔 vs 连续亏 3 笔，前者更极端

## 四、加仓/做T行为（±15分）

### 4.1 加仓效果（-6 ~ +8分）

**识别逻辑**：同 coin + 同方向，持仓未归零前再次 Open

**加重亏损型加仓**（正分，增加风险分）：
- 加仓后亏损扩大 >20% → +4分/次（上限+8分）
- 加仓后亏损扩大 >10% → +2分/次
- 加仓后亏损扩大 >0% → +1分/次

**摊薄成本型加仓**（负分，降低风险分）：
- 加仓后成功减亏且最终盈利 → -3分/次（下限-6分）
- 加仓后成功减亏但最终亏损 → -1分/次
- 加仓后无明显变化 → 0分

> 当前版本简化实现，返回 0 分，完整版需持仓状态机

---

### 4.2 做T行为（-3 ~ +7分）

**识别逻辑**：同 coin，持仓周期内 Open/Close 交替 ≥2次

**做T频率基础分**（+1~+3分）：
- 同周期开平 ≥4次 → +3分
- 同周期开平 ≥3次 → +2分
- 同周期开平 ≥2次 → +1分

**做T盈亏修正**（动态±分）：
- 做T累计盈亏为正，胜率>60% → -3分（有效策略）
- 做T累计盈亏为正，胜率≤60% → -1分
- 做T累计盈亏为负，亏损>总亏40% → +4分（做T反而放大亏损）
- 做T累计盈亏为负，亏损>总亏20% → +3分
- 做T累计盈亏为负，亏损<总亏10% → +1分

> 当前版本简化实现，返回 0 分

## 使用方式

```bash
# 1. 计算特征（v2）
python scripts/calculate_address_features_v2.py

# 2. 计算评分（v2）
python scripts/calculate_fragile_scores_v2.py

# 3. 查询结果
SELECT address, total_score, fragile_level
FROM hl_fragile_scores
ORDER BY total_score DESC
LIMIT 10;
```
